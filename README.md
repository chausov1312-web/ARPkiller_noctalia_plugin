# 🚀 ARP Killer – Noctalia Plugin

**Мощный ARP-spoofing инструмент в стиле Noctalia.**  
Позволяет выполнять MITM-атаки, полностью отключать интернет у выбранных устройств в локальной сети, а также использовать собственные команды.  
Встроенный автопилот: определение шлюза по IP/MAC, непрерывное сканирование сети, красивая панель с индикацией атаки, полностью настраиваемая иконка на панели.

---

## ✨ Возможности

- ✅ **Сканирование сети** – через `arp-scan` (список IP/MAC всех активных устройств)
- ✅ **Автоопределение шлюза** – IP через `ip route`, MAC через `ping` + `arp -n`
- ✅ **Непрерывное сканирование** – автоматическое обновление списка устройств (интервал от 1 секунды)
- ✅ **Три режима атаки**:
  - `MITM` – перехват трафика через ваш ПК (включается IP‑forwarding)
  - `KILL` – полное отключение интернета у жертвы (двойной спуфинг)
  - `CUSTOM` – своя команда с подстановкой переменных `{INTERFACE}`, `{VICTIM_IP}`, `{VICTIM_MAC}`, `{GATEWAY_IP}`, `{GATEWAY_MAC}`
- ✅ **Адаптивный интерфейс** – цвета и иконки следуют теме Noctalia
- ✅ **Индикация на панели** – количество атакуемых устройств, красный фон, выбор иконки и ширины кнопки
- ✅ **Отдельный список активных атак** – не пропадает при пересканировании
- ✅ **Автоматическое восстановление ARP** при остановке атаки (опционально)

## 📦 Зависимости (устанавливаются через пакетный менеджер)

```bash
sudo pacman -S arp-scan dsniff arping   # Arch / CachyOS
# Debian/Ubuntu:
sudo apt install arp-scan dsniff arping
```

---

## 🔐 Настройка `sudo` без пароля

Плагин вызывает системные утилиты с правами root. Добавьте следующую строку в файл `/etc/sudoers.d/arp-killer` (замените `user` на ваше имя пользователя):

```
user ALL=(ALL) NOPASSWD: /usr/bin/arp-scan, /usr/bin/arpspoof, /usr/bin/arping, /usr/bin/sysctl, /bin/ping
```

> Если `ping` находится в `/usr/bin/ping`, укажите полный путь: `which ping`.

Права на файл:
```bash
sudo chmod 440 /etc/sudoers.d/arp-killer
```

## 🛠 Установка плагина

1. Скопируйте папку `ARPkill` в директорию плагинов Noctalia:
   ```bash
   mkdir -p ~/.config/noctalia/plugins/ARPkill
   cp -r ARPkill/* ~/.config/noctalia/plugins/ARPkill/
   ```

2. Перезапустите Noctalia:
   ```bash
   qs -c noctalia-shell
   ```

3. Добавьте виджет `ARP Killer` на панель (через настройки виджетов).

## 🖱️ Использование

### Первый запуск

1. **Откройте настройки плагина** (правой кнопкой по иконке → Settings).
2. Выберите **сетевой интерфейс** (например, `wlan0` или `eth0`).
3. **(Опционально)** Нажмите **«Автоопределить шлюз»** – IP и MAC заполнятся автоматически.
4. Закройте настройки.

### Панель управления

Левой кнопкой по иконке открывается панель:

- **Шлюз** – можно редактировать вручную или нажать кнопку обновления.
- **Сканировать сеть** – обнаружит все устройства в вашей подсети.
- **Остановить все атаки** – завершит все активные спуфинг-процессы.
- **Непрерывное сканирование** – автоматически обновляет список устройств с заданным интервалом.
- **Активные атаки** – список IP/MAC, которые в данный момент атакуются. Кнопка остановки для каждого.
- **Обнаруженные устройства** – все найденные хосты. У атакуемых яркий фон и красный индикатор.

### Запуск / остановка атаки

Нажмите **«▶»** рядом с устройством → начнётся ARP‑спуфинг.  
Иконка на панели станет красной и покажет количество атакуемых устройств.  
Повторное нажатие на **«⏹»** остановит атаку и (если включено) восстановит ARP‑таблицу жертвы.

> **Важно:** Для MITM‑атаки трафик пойдёт через ваш ПК. Не забудьте включить IP‑forwarding (плагин сделает это автоматически при первом запуске атаки). Если вы хотите просто отключить интернет у жертвы, используйте режим `KILL`.

## ⚙️ Дополнительные настройки (Settings)

| Параметр | Описание |
|----------|----------|
| **Сетевой интерфейс** | Выбор активного интерфейса для сканирования и атак |
| **Шлюз по умолчанию** | IP и MAC роутера (можно оставить пустыми и нажать автоопределение) |
| **Тип атаки** | MITM, KILL или CUSTOM |
| **Пользовательская команда** (показывается только при выборе CUSTOM) | Произвольная команда с переменными `{INTERFACE}`, `{VICTIM_IP}`, `{VICTIM_MAC}`, `{GATEWAY_IP}`, `{GATEWAY_MAC}` |
| **Интервал сканирования (сек)** | От 1 до 300 секунд |
| **Автовосстановление ARP** | При остановке атаки отправляются корректные ARP-пакеты жертве и шлюзу |
| **Иконка в панели** | Любой текст/эмодзи (по умолчанию `⚡`) |
| **Ширина кнопки (px)** | Настройка размера виджета на панели |

## 🐞 Устранение неполадок

### MAC шлюза не определяется
- Убедитесь, что интерфейс выбран и сеть работает (`ping 8.8.8.8`).
- Проверьте права `sudo` на `/bin/ping` и `/usr/bin/arping`.
- Выполните вручную:
  ```bash
  ping -c 1 $(ip route show default | awk '{print $3}')
  arp -n
  ```
  Если MAC всё равно не отображается – проблема в ARP-кэше вашей системы (например, роутер не отвечает на ARP-запросы). Попробуйте указать MAC вручную в настройках.

### Атака не отключает интернет
- Для MITM обязательно должен быть включён IP‑forwarding (плагин включает его автоматически). Проверьте: `sysctl net.ipv4.ip_forward`.
- Если вы используете режим `KILL`, убедитесь, что `arpspoof` запускается с двумя командами (вторая подменяет шлюз для жертвы). При необходимости отредактируйте custom‑команду:
  ```bash
  sudo arpspoof -i {INTERFACE} -t {VICTIM_IP} {GATEWAY_IP} && sudo arpspoof -i {INTERFACE} -t {GATEWAY_IP} {VICTIM_IP}
  ```

### Не отображаются устройства при сканировании
- Убедитесь, что интерфейс указан верно (`ip link show`).
- Попробуйте запустить `sudo arp-scan --localnet --interface <iface>` вручную. Если нет вывода – возможно, arp-scan не установлен или нет прав.

### Ошибки в логах Noctalia
- Запустите Noctalia из терминала: `qs -c noctalia-shell` – все сообщения плагина будут видны.
- Ошибки вида `Cannot assign to non-existent property` обычно означают, что вы используете старую версию Noctalia (обновитесь).
